  <behavior>
    - Case 1 (no scope): init returns all active contexts in `context_documents`; additionally returns `task_scope: null` and `task_context_documents: []`.
    - Case 2 (scope = "planning"): init returns all active contexts in `context_documents`; additionally returns `task_scope: "planning"` and `task_context_documents` containing:
        - all unscoped context docs (no `extensions.task_scopes` key)
        - plus any context doc where `extensions.task_scopes` is a JSON array containing "planning"
    - Case 3 (scope mismatch): a context doc with `extensions.task_scopes: ["execution"]` is excluded from `task_context_documents` when scope="planning".
  </behavior>
  <action>
    In `crates/core/src/service.rs`:
    1) Add two fields to `InitBundleResult`:
       - `pub task_scope: Option<String>`
       - `pub task_context_documents: Vec<Document>`
       Keep existing fields (`governance`, `context_documents`) unchanged and still populated.

    2) Introduce a new helper entrypoint (keep the existing one for compatibility):
       - Add `pub async fn init_bundle_scoped(repo, governance, clock, ids, task_scope: Option<&str>) -> Result<InitBundleResult, CoreError>`.
       - Implement it by calling the existing governance reconcile path + `repo.list_active_context_documents()` once.
       - Populate `context_documents` with the full list (unchanged).
       - Populate `task_scope` and `task_context_documents` as follows:
         - If `task_scope` is None: `task_scope = None`, `task_context_documents = vec![]`.
         - If `task_scope` is Some(scope):
             - `task_scope = Some(scope.to_string())`
             - `task_context_documents = filter_task_contexts(&context_documents, scope)` where filtering rule is:
               * include doc if `doc.extensions` does NOT contain key `"task_scopes"`
               * OR doc.extensions["task_scopes"] is a JSON array of strings and contains `scope` exactly
               * otherwise exclude
             - Do NOT error if `task_scopes` exists but is not an array; treat as non-match (exclude from task_context_documents).

    3) Update existing `pub async fn init_bundle(...)` to call `init_bundle_scoped(..., None)` and return the result.

    4) Add unit tests in `crates/core/src/service.rs` verifying the three behavior cases above using `MemoryRepository`.
       - Create at least 3 context documents with distinct `extensions`:
         * unscoped: `{}`
         * planning: `{"task_scopes": ["planning"]}`
         * execution: `{"task_scopes": ["execution"]}`
       - Assert `context_documents.len()` remains 3 for both scoped and unscoped init.
       - Assert `task_context_documents` matches the expected subset.

    In `crates/core/src/lib.rs`:
    5) Export the new helper (`init_bundle_scoped`) alongside existing exports so interface crates can call it.
  </action>
