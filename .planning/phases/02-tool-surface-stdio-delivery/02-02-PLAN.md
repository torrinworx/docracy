    1) Update `crates/mcp/Cargo.toml` to support a binary entrypoint:
       - Add dependency: `clap = { version = "4", features = ["derive"] }`
       - Add dependency: `tracing = "0.1"`
       - Add dependency: `tracing-subscriber = { version = "0.3", features = ["env-filter"] }`
       - Upgrade `tokio` dependency features to include: `["macros", "rt", "rt-multi-thread", "sync", "io-std"]`
       - Add dev-dependencies: `assert_cmd = "2"`, `predicates = "3"`

    2) Create `crates/mcp/src/bin/docracy-mcp.rs` implementing a stdio MCP server:
       - Use `clap::Parser` for a minimal startup config:
         - `--database-url <string>` (optional; if absent, read `DATABASE_URL`; if still missing, exit 1)
         - `--governance-dir <path>` (default `./governance`)
         - `--no-migrate` (bool; if set, `run_migrations=false`)
       - Initialize tracing to stderr ONLY:
         - `tracing_subscriber::fmt().with_writer(std::io::stderr).with_env_filter(EnvFilter::from_default_env()).init();`
       - Build `docracy_mcp::McpStartupConfig` with `transport = McpTransport::Stdio`.
       - Bootstrap runtime via `docracy_mcp::bootstrap(&config).await`.
       - Construct tool service with `docracy_mcp::DocracyMcpServer::new(runtime)`.
       - Serve over stdio:
         - `let transport = rmcp::transport::stdio();`
         - `let server = service.serve(transport).await?;`
         - `server.waiting().await?;`

    3) Startup error discipline (CFG-03):
       - If bootstrap or serve fails before the MCP handshake completes:
         - Write a single-line JSON error envelope to **stderr** and exit with code 1.
         - Envelope format (exact keys): `{ "error": { "kind": "setup_error", "message": "..." } }`
       - Ensure **stdout is empty** in all failure paths (no `println!`, no `eprintln!` to stdout, no panics dumping to stdout).
  </action>
