  <action>
    1) Update `crates/mcp/Cargo.toml` dependencies:
       - Add `rmcp = { version = "1.3.0", features = ["server", "client", "transport-io", "transport-async-rw", "schemars"] }`
       - Add `schemars = { version = "1.0", features = ["derive"] }`
       - Add `tokio = { version = "1", features = ["sync"] }`

    2) Create `crates/mcp/src/tools.rs` implementing an rmcp server handler with tool router:
       - Define `#[derive(Clone)] pub struct DocracyMcpServer` with fields:
         - `tool_router: rmcp::handler::server::tool::ToolRouter<Self>`
         - `runtime: std::sync::Arc<tokio::sync::Mutex<Option<crate::runtime::McpRuntime>>>`
       - Provide constructors:
         - `pub fn new_unbootstrapped() -> Self` (runtime = None)
         - `pub fn new(runtime: crate::runtime::McpRuntime) -> Self` (runtime = Some(runtime))
       - Implement rmcp tool routing:
         - Add `#[rmcp::tool_router] impl DocracyMcpServer { ... }`
         - Implement exactly these tool functions (names MUST match):
           - `init()` with no params
           - `create(Parameters<CreateArgs>)`
           - `read(Parameters<ReadArgs>)`
           - `query(Parameters<QueryArgs>)`
           - `update(Parameters<UpdateArgs>)`
         - Each tool should return `Result<serde_json::Value, rmcp::model::ErrorData>`.

    3) Tool I/O shape MUST mirror the CLI JSON contract from `crates/cli/src/main.rs`:
       - `init` output: `{ "governance": {"files": [{"name": string, "content": string}]}, "context_documents": [...] }`
       - `create` input: `{ "type": string, "content": any, "extensions": object (default {}) }`
         output: `{ "document": <Document>, "revision": <DocumentRevision> }`
       - `read` input: `{ "ids": [uuid-string, ...] }`
         output: `{ "documents": [...], "missing_ids": [...] }`
       - `query` input: match core `QueryInput` fields exactly (including `where` key):
         `{ "query"?: string, "where"?: object, "order_by"?: [{"field": string, "direction": "asc"|"desc"}], "select"?: [string], "limit"?: number, "cursor"?: string }`
         output: `QueryResult` serialized to JSON (same as CLI)
       - `update` input: `{ "id": uuid-string, "expected_revision": uuid-string (accept alias "expected_head"), "content"?: any, "extensions"?: object, "status"?: string }`
         output: `{ "document": <Document>, "new_revision": <DocumentRevision>, "superseded_revision": <DocumentRevision> }`

    4) Convert tool args into core inputs using the same validation rules as the core/CLI:
       - For `create`: convert `type` string into `docracy_core::DocumentType::new(...)`; build `docracy_core::NewDocument` and call `validate()`; then call `crate::operations::create_document_runtime`.
       - For `query`: convert `QueryArgs` into `docracy_core::query::QueryInput` and call `crate::operations::query_documents_runtime`.
       - For `update`: parse `id` and `expected_revision` as UUIDs, wrap into `docracy_core::ids::DocumentId`/`RevisionId`, and call `crate::operations::update_document_runtime`.

    5) Error mapping (TOOL-03): extend `crates/mcp/src/error.rs` to convert `crate::error::McpError` into `rmcp::model::ErrorData`.
       - Add `impl McpError { pub fn to_error_data(&self) -> rmcp::model::ErrorData { ... } }`.
       - The produced `ErrorData.data` MUST be JSON: `{ "kind": <snake_case McpErrorKind>, "details": <details|null> }`.
       - Use this mapping for `ErrorData.code`:
         - `ValidationError` => `ErrorData::invalid_params(self.message.clone(), Some(data))`
         - `DocumentNotFound | RevisionNotFound | MissingCurrentRevision` => `ErrorData::invalid_request(self.message.clone(), Some(data))`
         - everything else => `ErrorData::internal_error(self.message.clone(), Some(data))`

    6) In `crates/mcp/src/lib.rs`, export the new server type:
       - `pub mod tools;`
       - `pub use tools::DocracyMcpServer;`

    7) Ensure every tool handler delegates ONLY through `crates/mcp/src/operations.rs` (no business rules added in tools).
