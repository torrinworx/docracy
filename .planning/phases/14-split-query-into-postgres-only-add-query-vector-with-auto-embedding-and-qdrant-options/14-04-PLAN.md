  <action>
    In `crates/mcp/src/tools.rs`:
    1) Update `QueryArgs`:
       - Remove `embedding: Option<Vec<f32>>`.
       - Keep the rest of the fields and `into_core()` mapping updated for the new core `QueryInput` shape.

    2) Add `QueryVectorArgs` with JSON schema fields:
       - `query: Option<String>`
       - `embedding: Option<Vec<f32>>`
       - `embed_model: Option<String>`
       - `where_` (rename "where", default)
       - `select: Vec<String>` (default)
       - `limit: Option<u32>`

    3) Add a new MCP tool handler method `query_vector` that:
       - Resolves embedding:
         - uses `embedding` if provided
         - else uses `query` (must be non-empty) and calls `docracy_postgres::ollama_embed_text(&query, embed_model.as_deref()).await`
         - else returns `ValidationError` via existing McpError mapping
       - Calls `docracy_core::query_vector_documents(&runtime.repo, QueryVectorInput { ... }).await`
       - Returns `Content::json(serde_json::to_value(out).unwrap())`

    In `crates/mcp/src/operations.rs`:
    - (Optional but preferred) add a thin helper `query_vector_documents(repo: &dyn Repository, input: QueryVectorInput)` that delegates to core, to match the existing `query_documents` pattern.

    In `crates/mcp/tests/operations.rs`:
    - Update the existing query delegation test to compile with the new `QueryInput` (no embedding field).
    - Add a small unit test that `QueryVectorArgs`-level validation rejects missing `query` and missing `embedding` (can be done by calling the tool handler directly if feasible; otherwise, add a helper-level validation function and test it).

    In `crates/mcp/README.md`:
    - Update the Tools list to include `query_vector`.
    - Update the `query` section to note it is Postgres-only.
    - Add a new `query_vector` section with input examples for:
      - explicit embedding
      - auto-embedding via query text
