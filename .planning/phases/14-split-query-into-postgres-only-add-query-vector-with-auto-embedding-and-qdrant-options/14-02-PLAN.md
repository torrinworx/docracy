  <behavior>
    - Given `input="hello"`, helper sends POST to `/api/embed` with `{"model": <model>, "input": "hello"}`.
    - Returns the first embedding vector (`embeddings[0]`) as `Vec<f32>`.
    - Rejects empty vectors and missing/invalid response shapes with a `RepoError::Storage(...)` message.
  </behavior>
  <action>
    In `crates/postgres/src/vector.rs`:
    - Add `pub async fn ollama_embed_text(input: &str, embed_model: Option<&str>) -> Result<Vec<f32>, RepoError>`.
    - Resolve configuration from env (same defaults as indexer):
      - `OLLAMA_URL` default `http://127.0.0.1:11434`
      - model = `embed_model.unwrap_or(env OLLAMA_EMBED_MODEL).unwrap_or("embeddinggemma")`
    - Implement request/response parsing exactly like the indexer (parse `embeddings[0]` array to floats).
    - Add a unit test using a local `TcpListener` mock server (pattern already used in this file) that:
      - captures the request path contains `POST /api/embed`
      - captures request body contains `"model":"..."` and `"input":"..."`
      - returns a fixed embedding and asserts the helper returns it.

    In `crates/postgres/src/lib.rs`:
    - Re-export the helper so downstream crates can use it: `pub use vector::ollama_embed_text;`.
  </action>

  <behavior>
    - Vector query filters are applied in Postgres; Qdrant provides ranking only.
    - Adapter requests more than `limit` candidates from Qdrant (bounded) to reduce post-filter dropouts.
    - Postgres integration tests that previously used `QueryInput.embedding` now use `QueryVectorInput` + `query_vector_documents`.
  </behavior>
  <action>
    In `crates/postgres/src/lib.rs` (`impl Repository for PgRepository`):
    - In `query_vector_documents(...)`:
      - Ensure Postgres filtering is not accidentally using full-text `content_search_tsv` by clearing `query.query` before calling `self.query_documents(...)`:
        - `let mut filter_query = query.clone(); filter_query.query = None;`
      - Change the Qdrant search limit to a bounded candidate multiplier:
        - `let requested = query.limit as usize;`
        - `let candidate_limit = (requested.saturating_mul(5)).clamp(requested, 100);`
        - Use `candidate_limit` for `qdrant_search_point_ids(...)`.

    In `crates/postgres/tests/postgres_integration.rs`:
    - Replace usages of `docracy_core::query_documents(&repo, QueryInput { embedding: Some(...) ... })` with the new vector query entrypoint:
      - Build `docracy_core::query::QueryVectorInput { embedding: <vec>, where_: <map>, select: <vec>, limit: <opt> }`
      - Call `docracy_core::query_vector_documents(&repo, input).await?` (import from `docracy_core` if re-exported).
    - Keep the same assertions for workspace scoping and archive exclusion.
  </action>
