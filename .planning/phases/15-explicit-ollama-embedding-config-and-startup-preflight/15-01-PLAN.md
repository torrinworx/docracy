<objective>
Make Ollama embedding model selection explicit inside the Postgres adapter and worker startup path, and add a verify-or-pull preflight so no embedding call guesses a model at runtime.

Purpose: remove hidden env/default drift between query, document-write, and worker embedding paths.
Output: explicit embed-model helper/preflight, repository-owned embed-model field, and worker startup validation.
</objective>

  <name>Task 1: Add explicit Ollama model helpers and preflight</name>
  <files>crates/postgres/src/vector.rs, crates/postgres/src/lib.rs</files>
  <read_first>
    - crates/postgres/src/vector.rs
    - crates/postgres/src/lib.rs

  <behavior>
    - `ollama_embed_text` takes `embed_model: &str` and never reads `OLLAMA_EMBED_MODEL` itself.
    - `require_ollama_embed_model` rejects missing or blank startup config values instead of inventing a default.
    - `verify_or_pull_ollama_embed_model` first calls `POST /api/show` and falls back to `POST /api/pull` with `{"model": ..., "stream": false}` on a missing-model response.
  </behavior>
  <action>
    In `crates/postgres/src/vector.rs`:
    - Change `ollama_embed_text` to require an explicit `&str` model argument.
    - Add `pub fn require_ollama_embed_model(raw: Option<String>) -> Result<String, RepoError>` that trims whitespace and returns `RepoError::Storage("OLLAMA_EMBED_MODEL is required")` when the value is missing or blank.
    - Add `pub async fn verify_or_pull_ollama_embed_model(ollama_url: &str, embed_model: &str) -> Result<(), RepoError>` that:
      1) POSTs `/api/show` with `{"model": embed_model}`
      2) returns `Ok(())` when the model exists
      3) on 404, POSTs `/api/pull` with `{"model": embed_model, "stream": false}`
      4) surfaces any other status as `RepoError::Storage(...)`
    - Add local `TcpListener` tests that capture the request paths/bodies for both the show-success and show-missing/pull paths.

    In `crates/postgres/src/lib.rs`:
    - Re-export `require_ollama_embed_model` and `verify_or_pull_ollama_embed_model` alongside `ollama_embed_text`.
  </action>
  <acceptance_criteria>
    - `crates/postgres/src/vector.rs` no longer has an `Option<&str>` parameter on `ollama_embed_text`.
    - `crates/postgres/src/vector.rs` contains both `require_ollama_embed_model` and `verify_or_pull_ollama_embed_model`.
    - `cargo test -p docracy-postgres ollama -- --nocapture` passes.
  </acceptance_criteria>
  <verify>
    <automated>cargo test -p docracy-postgres ollama -- --nocapture</automated>
  </verify>
