  <behavior>
    - Canonical document/revision writes happen first; the mirror queue records only the latest current snapshot.
    - Rows are keyed by workspace + document so workspace scopes cannot contaminate one another.
    - Archived/deleted state changes replace the current queued snapshot rather than appending stale history.
  </behavior>
  <action>
    Add a new `vector_mirror_queue` table in `migrations/0007_vector_mirror_queue.sql` that stores the latest embeddable snapshot per `(workspace_id, document_id)`, including the current revision id, archive/delete timestamps, and the embedding payload plus its dimension. In `crates/postgres/src/lib.rs`, enqueue or refresh the row inside the same transaction that creates or updates a document/revision, using the session workspace id that already scopes the adapter. Keep the queue current-only: later writes overwrite the prior row for the same workspace/document pair. If a document has no embedding payload, leave the canonical Postgres write unchanged and skip queueing.
  </action>
  <acceptance_criteria>
    - `migrations/0007_vector_mirror_queue.sql` contains `vector_mirror_queue`.
    - `crates/postgres/src/lib.rs` contains a helper that enqueues mirror snapshots from document writes.
    - Queue rows are scoped by workspace id and document id.
  </acceptance_criteria>
  <verify>
    <automated>cargo test -p docracy-postgres --test postgres_integration vector_mirror_queue</automated>
  </verify>
  <done>
    Postgres now records the latest mirror snapshot in a scoped queue whenever the canonical document state changes.</done>
