## Problem

Docracy needs a workflow document that can mirror some of the useful capture and planning behavior from the GSD addon, but the exact shape should be informed by how `get-shit-done` actually works. The upstream project is a runtime-installed workflow layer with staged commands, persistent planning artifacts, todo capture, and state tracking, so the local workflow doc should reflect those patterns instead of inventing a new one from scratch.

## Solution

Research the GSD repo and its command/artifact model first, then draft a workflow document that captures the subset of behavior Docracy needs: staged capture, persistent planning context, todo/backlog handling, and any state/commit conventions that fit this project.
